### 3.3 核心工作流程（补充技术细节）
#### 流程1：需求输入与解析
1. 用户在SolidWorks插件界面输入自然语言需求（如“前视基准面拉伸200x100x20底座，四角打4个直径10通孔”）；
2. 客户端`UI交互模块`校验输入合法性（非空、无敏感字符），并收集环境信息（SolidWorks版本、当前文档类型）；
3. 客户端`网络通信模块`将需求与环境信息封装为JSON格式，通过HTTP POST发送至服务端（本地部署时地址为`http://127.0.0.1:8000/api/generate`）。

#### 流程2：AI指令生成
1. 服务端`请求处理模块`解析JSON数据，通过`提示词工程模块`进行需求结构化：
   - 提取关键信息：基准面（前视基准面）、特征1（拉伸凸台：200x100x20）、特征2（切除：4个圆形孔，直径10，完全贯穿）；
   - 补充领域术语：将“四角”映射为“矩形四个顶点偏移10mm”，“通孔”映射为“完全贯穿切除”；
   - 生成标准化提示词：`"基于SolidWorks 2025，在前视基准面创建200mm×100mm矩形草图，拉伸20mm生成凸台；在上视基准面，以矩形四角为中心（偏移10mm）创建4个直径10mm圆形草图，执行完全贯穿切除，生成通孔。输出JSON格式指令，包含特征类型、参数、顺序，参数需符合SolidWorks COM API要求。"`
2. 服务端`LLM接口适配层`调用指定AI引擎（默认GPT-4o），获取建模指令。

#### 流程3：指令执行与反馈
1. 服务端`指令标准化模块`校验AI返回结果，确保格式正确（示例如下）：
```json
{
  "features": [
    {
      "feature_type": "extrude",
      "params": {
        "plane": "前视基准面",
        "shape": "rectangle",
        "length": 200,
        "width": 100,
        "depth": 20,
        "direction": "positive"
      }
    },
    {
      "feature_type": "cut",
      "params": {
        "plane": "上视基准面",
        "shape": "circle",
        "diameter": 10,
        "count": 4,
        "layout": "four_corner",
        "offset": 10,
        "through_all": true
      }
    }
  ]
}
```
2. 客户端`SolidWorks适配模块`解析指令，通过COM API执行操作：
   - 激活指定基准面→绘制草图（调用`SketchManager`）→执行特征（调用`FeatureManager`）；
   - 实时监听执行状态，若出现“草图超出实体”“参数无效”等错误，触发服务端重试逻辑；
3. 建模完成后，客户端展示结果日志（如“拉伸凸台成功→4个通孔生成成功”），支持用户点击“参数微调”修改尺寸。

## 四、核心功能详细说明
### 4.1 核心功能1：自然语言建模（P0级）
#### 4.1.1 功能逻辑拆解
1. **需求识别层**：基于领域术语库（含100+工程设计关键词），通过正则匹配+LLM语义理解，提取基准面、形状、尺寸、特征类型等关键信息；
2. **指令生成层**：将自然语言映射为SolidWorks可执行的API参数（如“拉伸”→`FeatureExtrusion3`，“完全贯穿”→`swEndConditions_e.swEndCondThroughAll`）；
3. **执行层**：通过COM API调用SolidWorks底层功能，自动完成草图绘制、特征生成，无需人工点击。

#### 4.1.2 支持的建模元素
| 类别 | 具体元素 | 技术实现细节 |
|------|----------|--------------|
| 基准面 | 前视、上视、右视基准面 | 自动识别基准面名称，调用`ModelDoc2.Extension.SelectByID2`选中 |
| 草图形状 | 矩形、圆形、槽口、圆弧 | 调用`SketchManager`的`CreateCornerRectangle`/`CreateCircle`等方法 |
| 特征操作 | 拉伸凸台、拉伸切除 | 调用`FeatureManager.FeatureExtrusion3`（凸台）/`FeatureCut4`（切除） |
| 终止条件 | 给定深度、完全贯穿 | 动态设置`T1`参数（`swEndCondBlind`/`swEndCondThroughAll`） |

#### 4.1.3 典型使用案例（含技术参数）
| 输入需求 | 系统解析结果 | API调用关键参数 | 执行耗时 |
|----------|--------------|----------------|----------|
| “前视基准面画150x80矩形，拉伸30mm” | 凸台拉伸：前视基准面，矩形（150×80），深度30mm | `FeatureExtrusion3(Sd=true, T1=swEndCondBlind, D1=0.03)` | 3.2秒 |
| “上视基准面打1个直径8mm的孔，中心坐标(50,50)，完全贯穿” | 切除：上视基准面，圆形（直径8），中心(50,50)，贯穿 | `FeatureCut4(T1=swEndCondThroughAll, D1=0)` | 2.8秒 |
| “前视基准面拉伸200x100x20底座，四角各打1个直径10mm通孔，距离边缘10mm” | 凸台+4个切除：孔位坐标自动计算（(90,40)、(90,-40)、(-90,40)、(-90,-40)） | 循环调用`FeatureCut4`，批量生成孔特征 | 12.5秒 |

### 4.2 核心功能2：孔位自动布局（P1级）
#### 4.2.1 布局算法设计
针对“四角孔”“矩阵孔”等常见需求，设计几何计算算法：
1. **四角孔布局**：根据底座长度（L）、宽度（W）、边缘偏移（O），自动计算4个孔中心坐标：
   - 右上：(L/2 - O, W/2 - O)
   - 右下：(L/2 - O, -W/2 + O)
   - 左上：(-L/2 + O, W/2 - O)
   - 左下：(-L/2 + O, -W/2 + O)
2. **矩阵孔布局**：支持用户输入“行数×列数”（如3×4），自动计算孔间距，避免孔位重叠或超出实体范围。

#### 4.2.2 动态适配逻辑
若用户未指定边缘偏移，默认设置为“孔直径的1倍”（如直径10mm的孔，偏移10mm）；若计算出的孔位超出实体范围，自动调整偏移值至最小合理值（≥5mm）。

### 4.3 核心功能3：插件式客户端（P0级）
#### 4.3.1 客户端界面设计
采用轻量化WPF界面，嵌入SolidWorks插件栏，界面包含3个核心区域：
1. 需求输入区：多行文本框，支持粘贴需求文本，带“清空”“提交”按钮；
2. 状态展示区：实时显示建模进度（如“正在绘制草图→正在执行拉伸”），带进度条；
3. 参数微调区：建模完成后，可修改核心尺寸（如拉伸深度、孔直径），点击“重新执行”快速更新模型。

### 4.4 核心功能4：多LLM兼容（P1级）
#### 4.4.1 接口适配层设计
抽象`LLMClient`基类，不同模型实现子类，通过配置文件切换，核心代码结构：
```python
# 基类：统一接口
class LLMClient:
    def __init__(self, api_key, base_url):
        self.api_key = api_key
        self.base_url = base_url

    def generate_command(self, prompt):
        raise NotImplementedError()

# OpenAI实现类
class OpenAIClient(LLMClient):
    def generate_command(self, prompt):
        client = OpenAI(api_key=self.api_key, base_url=self.base_url)
        response = client.chat.completions.create(
            model="gpt-4o",
            messages=[{"role": "user", "content": prompt}]
        )
        return response.choices[0].message.content

# DeepSeek实现类
class DeepSeekClient(LLMClient):
    def generate_command(self, prompt):
        client = DeepSeekChat(api_key=self.api_key, base_url=self.base_url)
        response = client.chat.completions.create(
            model="deepseek-chat",
            messages=[{"role": "user", "content": prompt}]
        )
        return response.choices[0].message.content
```

### 4.5 扩展功能：参数微调与模型导出（P2级）
#### 4.5.1 参数微调
建模完成后，客户端自动提取可修改参数（如尺寸、孔数、偏移量），用户输入新值后，系统仅重新执行相关特征，无需从头建模（如修改孔直径从10mm为8mm，仅重新执行切除特征，耗时≤2秒）。

    );
}

### 5.1 技术创新1：工程领域指令精准生成机制
#### 创新点描述
解决通用LLM对工程术语识别不准确、指令格式不兼容SolidWorks API的问题，通过“**术语库映射+提示词模板+指令校验**”三层机制，将自然语言需求转化为高准确率的建模指令。

#### 技术实现细节
1. **领域术语库**：手动整理100+工程设计核心术语（如“基准面”“完全贯穿”“圆角”），建立“自然语言→API参数”映射表（如“四角孔”→`layout: four_corner`）；
2. **提示词模板优化**：针对SolidWorks建模场景定制提示词模板，包含“需求结构化提取→参数格式校验→API兼容性适配”三个环节，示例模板：
```
你是SolidWorks建模专家，请按以下步骤处理用户需求：
1. 提取关键信息：基准面（前视/上视/右视）、形状（矩形/圆形/槽口）、尺寸（长/宽/深/直径）、特征类型（拉伸/切除）、终止条件（完全贯穿/给定深度）；
2. 生成JSON格式指令，参数需符合SolidWorks COM API要求：
   - 拉伸特征使用FeatureExtrusion3，切除特征使用FeatureCut4；
   - 终止条件对应swEndConditions_e枚举（完全贯穿=swEndCondThroughAll，给定深度=swEndCondBlind）；
3. 若需求信息不全，默认补充：边缘偏移=10mm，拉伸方向=正方向，无圆角；
4. 禁止添加额外字段，确保JSON可直接解析执行。
```
3. **指令校验层**：服务端通过正则表达式校验AI返回的JSON指令，确保特征类型、参数名称、数值范围符合要求（如深度>0，直径>0），校验失败则触发AI重试。

#### 效果数据
通用LLM直接生成指令准确率仅65%，通过本机制优化后，准确率提升至92%，复杂需求（多特征组合）准确率达85%。

### 5.2 技术创新2：SolidWorks多版本自适应机制
#### 创新点描述
针对SolidWorks 2020-2025版本API参数差异（如`FeatureExtrusion2`与`FeatureExtrusion3`的参数个数不同），设计动态适配逻辑，无需用户手动切换版本配置。

#### 技术实现细节
1. **版本识别**：客户端启动时调用`_swApp.RevisionNumber`获取SolidWorks版本号；
2. **API参数适配**：通过条件判断动态调整API参数，示例代码：
```csharp
IFeatureManager featureManager = _swApp.FeatureManager;
if (swVersion >= 2023)
{
    // 2023+版本使用FeatureExtrusion3
    featureManager.FeatureExtrusion3(Sd: true, Flip: false, Dir: false, ...);
}
else
{
    // 2020-2022版本使用FeatureExtrusion2
    featureManager.FeatureExtrusion2(Sd: true, Flip: false, Dir: false, ...);
}
```
3. **兼容性测试**：在SolidWorks 2020、2023、2025三个主流版本中完成100组测试用例，无功能缺失或报错。

### 5.3 技术创新3：个人开发轻量化部署方案
#### 创新点描述
针对个人开发场景，设计“**客户端一键安装+服务端单文件运行**”的部署方案，无需复杂环境配置，用户可在5分钟内完成部署并使用。

#### 技术实现细节
1. **客户端打包**：使用Inno Setup将C#插件、依赖DLL、注册脚本打包为EXE安装包，用户双击安装后自动完成SolidWorks插件注册；
2. **服务端打包**：使用PyInstaller将Python服务端、依赖库打包为单EXE文件（体积≤50MB），双击即可启动服务（默认端口8000）；
3. **零依赖运行**：服务端打包时嵌入Python解释器，用户无需安装Python环境；客户端依赖自动注册，无需手动配置COM组件。

## 八、部署与使用说明
### 8.1 部署环境要求
| 环境类型 | 硬件要求 | 软件要求 |
|----------|----------|----------|
| 客户端（建模端） | CPU≥i5，内存≥8GB，硬盘≥10GB | Windows 10/11 64位，SolidWorks 2020-2025 |
| 服务端（本地部署） | CPU≥i3，内存≥4GB | Windows 10/11 64位（无需SolidWorks） |

### 8.2 部署步骤（个人用户，5分钟完成）
#### 步骤1：服务端部署
1. 下载服务端打包文件`VulcanServer.exe`；
2. 双击运行`VulcanServer.exe`，看到“Uvicorn running on http://127.0.0.1:8000”即启动成功；
3. （可选）修改`config.env`文件切换LLM模型（需填入API密钥）。

#### 步骤2：客户端部署
1. 下载客户端安装包`VulcanClient_Setup.exe`；
2. 双击安装，按提示完成下一步（默认安装路径：`C:\Program Files\Vulcan`）；
3. 打开SolidWorks，在“工具→插件”中勾选“Vulcan AI建模助手”，点击“确定”。

### 8.3 使用步骤（3步上手）
1. 打开SolidWorks，新建零件文档；
2. 在SolidWorks插件栏点击“Vulcan”，打开操作面板；
3. 输入建模需求（如“前视基准面拉伸200x100x20底座，四角打4个直径10mm通孔”），点击“提交”，等待建模完成。

### 8.4 常见问题排查
| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 插件未在SolidWorks中显示 | 安装时未以管理员身份运行 | 卸载后，右键以管理员身份重新安装 |
| 服务端启动失败 | 端口8000被占用 | 修改`config.env`中PORT为其他端口（如8080），同步修改客户端`ApiClient.cs`中的地址 |
| 建模时报“草图无效” | 需求中孔位/形状超出实体范围 | 增加边缘偏移（如“距离边缘15mm”），重新提交 |
| LLM调用失败 | API密钥无效或网络问题 | 检查API密钥，切换网络或LLM模型（如DeepSeek） |
